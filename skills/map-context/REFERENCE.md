# map-context Reference

## Qualifying Directories

A directory gets a CONTEXT.md if it has **3+ substantive files** with real logic. Skip if:

- Fewer than 3 substantive files (excludes config, lock files, assets, test fixtures)
- Pure type/config/asset directory — no business logic
- Trivially named: `scripts/`, `migrations/`, `__pycache__/`, `.git/`, `node_modules/`, `dist/`, `build/`, `public/`, `static/`

Always print skip reasoning:

```
skipping src/types — only type declarations, no logic
skipping src/migrations — database migrations, not application logic
```

## Depth Rule

Recurse fully, but **flatten by default**: document `src/auth/` as one CONTEXT.md even if it contains `src/auth/providers/`. Give a subdirectory its own CONTEXT.md only if:

- It passes the 3-file heuristic independently, AND
- It has a clearly distinct responsibility from its parent

## CONTEXT.md Content

Write in prose. Cover:

- **Key abstractions** — what the main files/classes/functions do and WHY they exist
- **Data flows** — what goes in, what comes out, what side effects happen
- **Integration points** — what this section depends on and what depends on it
- **Non-obvious constraints or gotchas**

No file listings — those go stale and are derivable from `ls`.

## CONTEXT-MAP.md Format

Lives at the **repo root**. Keep under 60 lines.

```markdown
# Context Map

> Generated by /map-context. Read this first to orient yourself before exploring the codebase.

## path/to/dir
One sentence description of what this section does.
→ path/to/dir/CONTEXT.md

## path/to/dir
One sentence description.
→ path/to/dir/CONTEXT.md
```

## CLAUDE.md / AGENTS.md Block

Add or replace the `## Codebase Context` section with:

```markdown
## Codebase Context

Before exploring the codebase, read `CONTEXT-MAP.md` at the repo root.
It maps each section to a `CONTEXT.md` file with abstractions, data flows,
and integration points. Load the relevant CONTEXT.md files instead of
reading source files directly.
```

**CLAUDE.md**: Create if missing. If a `## Codebase Context` section already exists, replace it. Otherwise append.

**AGENTS.md**: Same block, same logic. Never create — only update if the file already exists.
